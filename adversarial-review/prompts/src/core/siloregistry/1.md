# Mission 1 — The miswired-silo adversary (can an inconsistent silo earn senior backing?)

MISSION: 1 — the 6-clause topology web + admission guards (I-1, I-2, I-3, I-11)

## Who you are
You attack the underwriting gate: the claim that `addSilo` admits ONLY a self-consistent silo — one whose
freeze/escrow/coordinator/adapter all point ONLY at its OWN pool/safe/oracle. Your goal: ONE silo that points
ANY component at a DIFFERENT (a neighbor's) pool/safe/oracle slot and slips through, a topology clause that
doesn't actually bind, or a zero cfg field that passes the zero-address sweep.

## The claims you are attacking
- **I-1 (the 6-clause topology web).** `addSilo` (`:164`, onlyOwner) reverts `SiloMiswired` (`:177-183`) unless
  ALL SIX clauses hold. Read each clause against the getter it dereferences and confirm it binds the silo's
  OWN component, not a neighbor's:
  1. `IFreeze(cfg.freeze).eulerEarn() == cfg.eePool` (`:178`) — getter `DurationFreezeModule.eulerEarn()`
     (`contracts/src/supply/szipUSD/DurationFreezeModule.sol:57`).
  2. `IFreeze(cfg.freeze).warehouseSafe() == cfg.warehouseSafe` (`:178`) — `DurationFreezeModule.warehouseSafe()`
     (`DurationFreezeModule.sol:59`).
  3. `IFreeze(cfg.freeze).navOracle() == cfg.navOracle` (`:179`) — `DurationFreezeModule.navOracle()`
     (`DurationFreezeModule.sol:54`).
  4. `IEscrow(cfg.escrow).coordinator() == cfg.defaultCoordinator` (`:180`) — `LienXAlphaEscrow.coordinator()`
     (`contracts/src/loss/LienXAlphaEscrow.sol:61`).
  5. `INavWriter(cfg.defaultCoordinator).navOracle() == cfg.navOracle` (`:181`) — `DefaultCoordinator.navOracle()`
     (`contracts/src/loss/DefaultCoordinator.sol:90`).
  6. `ISeniorVenue(cfg.adapter).seniorPool() == cfg.eePool` (`:182`) — `EulerVenueAdapter.seniorPool()`
     (`contracts/src/venue/EulerVenueAdapter.sol:522`).
  **Find a clause that's satisfiable with a cross-pointer** — e.g. a silo whose `freeze` points its `eulerEarn`
  at silo-B's pool while `cfg.eePool` is also silo-B's pool (the clause holds but the silo reads a foreign
  surface) — or a component that is reachable but NOT asserted (`curator`/`juniorBasket` are carried for
  routing only, `:176`/`:184` — confirm omitting them from the assert is intentional, not a missed clause).
- **I-2 (admission guards).** `ZeroSiloId` (`:165`, zero is the reserved `currentSilo` sentinel),
  `DuplicateSilo` (`:166`, re-key of an existing adapter), `ZeroAddress` (`:169-173`) on ANY of the **9** cfg
  address fields. **Find a zero field that escapes the sweep** — read the `||` chain `:170-172` and confirm all
  nine fields (`adapter`, `warehouseSafe`, `eePool`, `juniorBasket`, `escrow`, `defaultCoordinator`,
  `navOracle`, `freeze`, `curator`) are checked, none dropped.
- **I-3 (happy admission + currentSilo adoption).** On success (`:185-203`): writes the `Silo` with `cfg`'s
  addresses + `lineCount=0` + `active=true` (`:195-196`), pushes to `siloIds` (`:198`), and adopts as
  `currentSilo` ONLY if none is set (`:199-202`). Confirm the first silo becomes `currentSilo` and a second
  does NOT silently steal it.
- **I-11 (CTR-10b venue-agnostic).** The gate dereferences only the venue-neutral `ISeniorVenue.seniorPool()`
  + the `ISeniorPool`-shaped `IFreeze` slots, so a non-Euler venue plugs in unchanged. Confirm no
  Euler-specific surface is dereferenced. (The donation-immunity of a real non-Euler senior surface is that
  venue's own property — out of scope here, correctly noted in NatSpec `:71-73`.)

## Specific attacks to run
1. **The cross-pointer.** Build a `cfg` where every clause holds numerically but the silo reads a neighbor's
   pool — does any clause fail to anchor `cfg.eePool` / `cfg.navOracle` to the silo's OWN surface? Trace
   whether the six clauses TRANSITIVELY pin one self-consistent `(eePool, warehouseSafe, navOracle,
   defaultCoordinator)` set, or whether a curator could satisfy all six with a Frankenstein silo.
2. **The unasserted fields.** `juniorBasket` (`:189`) and `curator` (`:194`) are stored but NOT in the assert.
   Confirm that's by design (routing/labeling only) and that nothing safety-critical reads them as if asserted.
3. **The zero sweep completeness.** Read `:169-173` literally — count the nine `== address(0)` disjuncts.
   A missing field would let a zero address into a stored silo (and the topology assert would then revert on
   the zero component anyway for asserted fields — but `juniorBasket`/`curator` aren't asserted, so a zero
   there would only be caught by the sweep).
4. **Reentrancy via a hostile getter.** The six clauses call out to `cfg.freeze`/`cfg.escrow`/
   `cfg.defaultCoordinator`/`cfg.adapter` (attacker-supplied addresses, since the curator chooses them) BEFORE
   the storage write (`:185`). Could a hostile getter re-enter `addSilo` (or another owner fn) mid-assert?
   `addSilo` is `onlyOwner` (Timelock) — the cfg components are vetted by the Timelock's own underwriting, and
   there is no state written before the external calls. Frame: a hostile getter griefs the Timelock's own tx,
   not an outsider attack; rate accordingly.

The 6 `test_addSilo_miswired_{freezeEePool,escrowCoordinator,freezeWarehouse,freezeNavOracle,
coordinatorNavOracle,adapterEePool}_reverts` (each clause negated), `test_addSilo_zeroId_reverts`,
`_duplicate_reverts`, `_zeroAddress_eachField_reverts` (loops all 9), `_happyPath`, `_currentSilo_onlyFirst`.
Confirming the 6-clause web binds + the guards hold goes in your Summary. A finding must be a concrete
miswired-silo-admitted / clause-doesn't-bind / zero-field-escapes path the negation tests do NOT cover — most
valuably a cross-pointer the six clauses fail to transitively pin to one self-consistent surface.
