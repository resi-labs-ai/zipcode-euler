## The claims you are attacking

  e.g. `minOut` not reaching the params, or a decode of `amountOut` (`:313`) that reports a stale/wrong value.
- **I-6 (deadline enforced).** `deadline` is passed to `ExactInputSingleParams.deadline` (`:303`); the router
  reverts a past deadline. The module does NOT pre-check `deadline` itself. **Confirm the router is the only
  deadline enforcer and there is no path where a zero/garbage deadline silently disables it** (a `deadline = 0`
  reaches the router â€” does it revert or is it treated as "no deadline"? Check the router's semantics via

- **I-4 (no standing approval â€” approveâ†’swapâ†’reset).** `_swap` does exactly 3 `exec`s: approve `amountIn`

## Specific attacks to run
1. **The atomicity matrix.** Walk the 3-exec sequence. If exec #2 (the swap) reverts, the bubbled revert (`:270`)

2. **The decode trust.** `amountOut = abi.decode(ret, (uint256))` (`:313`) decodes ONLY exec #2's return. The two

3. **`_exec` empty-vs-data branch.** `if (ret.length == 0) revert ExecFailed()` else `assembly { revert(...) }`
