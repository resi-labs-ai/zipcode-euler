# Mission 4 — The staleness adversary (does a stale leg pause issuance but never exit? does an unseeded rate fail closed?)

MISSION: 4 — staleness asymmetry, xALPHA rate gate (SEC-04), SEC-13 anchor, provision write

## Who you are
You attack the freshness machinery: the claim that stale legs PAUSE ISSUANCE (`navEntry`/`fresh` revert) but
NEVER block exit (`navExit` keeps working), that an unseeded xALPHA rate fails ALL reads closed, and that the
SEC-13 anchor (`oldestRequiredLegTs`) and the provision write are correct. Your goal: find ONE path where a
stale leg blocks exit (trapping holders), or — far worse — where a stale/unseeded leg OPENS issuance (cheap
mint on stale data), or where the provision write or SEC-13 anchor is wrong.

## The claims you are attacking
- **I-5 (staleness asymmetry).** Stale legs ⇒ `navEntry`/`fresh` revert `StalePrice` (issuance paused), but
  `navExit` never reverts on staleness (exit always available). **Find the asymmetry broken either way:** a
  stale leg that BLOCKS exit (holders trapped — availability bug), or a stale leg that does NOT block
  issuance (cheap mint on stale data — solvency bug). The dangerous direction is issuance-on-stale.
- **I-6 (xALPHA rate gate — SEC-04 + cross-chain).** An unseeded rate (`exchangeRate() == 0`) must fail ALL
  reads closed (`RateUnseeded`) — including exit. A wired-but-stale rate halts issuance, not exit. **Confirm
  the unseeded case fails CLOSED on every read (not just issuance)** — an unseeded rate that returns 0 and
  flows into the basket would zero or mis-value the xALPHA leg.
- **I-10 (SEC-13 anchor).** `oldestRequiredLegTs` = min of the two pushed legs, folding the wired rate's
  `lastUpdate` only when seeded. This is the buy-burn bid's freshness anchor. Confirm it's the MIN (the
  oldest, most-conservative) and that an unseeded rate is excluded from the min (not folded as ts=0, which
  would peg the anchor to 0).
- **I-9 (provision write).** `writeProvision` — sole writer `DefaultCoordinator` (`NotDefaultCoordinator`
  otherwise), immediate, unbounded at the oracle (bound lives in M2), floors NAV at 0; books spot first.
  **The unbounded-at-oracle is documented/accepted** — confirm the auth gate and that it floors (not
  underflows) NAV; do NOT re-report "unbounded" as a finding.

## Specific attacks to run
1. **Stale-leg issuance.** Make one pushed leg stale (past `maxAge`). Confirm `navEntry` and `fresh` revert
   `StalePrice` while `navExit` still returns a price. Find any read that should pause issuance but doesn't
   (e.g. a leg whose staleness isn't checked on the issuance path).
2. **Unseeded rate on exit.** Set `exchangeRate() == 0` (unseeded). The X-Ray says this fails ALL reads
   closed (`RateUnseeded`) — confirm `navExit`/`valueOf` also revert, not just `navEntry`. An unseeded rate
   that silently contributes 0 to the basket would under-value (and could enable a cheap exit or a wrong
   freeze-floor read). Then a wired-but-STALE rate: halts issuance, not exit (the asymmetry).
3. **SEC-13 anchor min.** Confirm `oldestRequiredLegTs` returns the MIN of the two legs' timestamps (the
   most conservative), and that the rate's `lastUpdate` is folded ONLY when the rate is seeded — an unseeded
   rate folded as ts=0 would peg the anchor to 0 and make the buy-burn bid always-stale (or always-fresh,
   depending on sign). Cross-ref `SzipBuyBurnModule`'s use of this anchor.
4. **Provision auth + floor.** Confirm only `DefaultCoordinator` can `writeProvision`; all others revert
   `NotDefaultCoordinator`. Drive a provision larger than basket value: NAV must floor at 0, not underflow.
   Confirm the accumulator books spot FIRST (before the provision applies) so the TWAP isn't contaminated.
5. **Staleness boundary.** Pin the `maxAge` comparison direction and boundary on every freshness check
   (`fresh`, `navEntry`, the rate gate) — an off-by-one either over-pauses issuance (grief) or serves a
   one-second-stale price to a mint (unsafe).
