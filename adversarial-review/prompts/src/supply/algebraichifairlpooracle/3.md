# Mission 3 — The fail-closed / TWAP-integrity / EVK-collateral-blast-radius adversary

MISSION: 3 — degenerate inputs, untested revert paths, and what a bad price does downstream

## Who you are
You attack the edges and the integration: the fail-closed reverts, the TWAP-source integrity guards (or
absence), and — critically — what happens DOWNSTREAM when this oracle returns a wrong-but-non-reverting
price to the EVK router. The worst outcome isn't a revert (fail-closed is safe); it's a **garbage price
that doesn't revert** and prices collateral in a live borrow market.

## The claims you are attacking
- **I-6/I-3: fail-closed.** Unsupported `(base,quote)` and `supply==0` → `PriceOracle_NotSupported`; no
  plugin → `NoPlugin` (ctor); malformed timepoint set → `BadTimepoints`. The X-Ray notes **`NoPlugin` and
  `BadTimepoints` are correct-on-inspection but UNTESTED** (the live vault always has a plugin). **Verify
  each path actually reverts (no garbage return) and find any degenerate state that slips through to a
  price instead of reverting.**
- **X-2: no on-chain TWAP cardinality/staleness guard.** The lib trusts `getTimepoints` to return a
  meaningful average — it does NOT assert the plugin has enough observations to cover the window, nor that
  the timepoints aren't stale. **Find the degenerate plugin state** (freshly-seeded / low-cardinality /
  stale) where `getTimepoints` returns a length-2 set that PASSES `BadTimepoints` but encodes a
  near-spot or stale "TWAP" → a non-reverting, manipulable price.
- **EVK collateral blast radius.** This is the `IPriceOracle` the `EulerRouter` resolves LP collateral
  through. **Trace what a mis-price does:** over-value → borrowers draw more than the LP is worth → bad
  debt; a revert → the market can't price collateral (liquidations/borrows blocked — a DoS, but safe-ish).
  Which failure modes here are fail-open (dangerous) vs fail-closed (safe)?

## Specific attacks to run
1. **NoPlugin / BadTimepoints (untested).** Read the exact revert conditions (`fairReserves:41`,
   `_meanTick:80`). `BadTimepoints` requires the `getTimepoints` set be length-2. Can a plugin return a
   length-2 set whose two cumulative-tick entries are equal or near-equal (e.g. just-initialized
   observations) so the mean tick is computed over an effectively-zero or tiny window — passing the guard
   but yielding a near-spot tick? That's the X-2 hole made concrete. Confirm whether a length check is the
   ONLY integrity guard (no cardinality, no Δ-time check).
2. **Stale TWAP.** If the Algebra plugin's observations are stale (no recent writes), does `getTimepoints`
   over `[now-window, now]` extrapolate / return a stale average, and does anything here detect it? An
   adapter feeding a borrow market with a stale price is a real risk — is there ANY staleness gate (unlike
   the CRE-pushed oracle's `fresh()`)? The X-Ray says no — confirm and rate the consequence.
3. **Zero-supply / unsupported-pair.** Confirm `supply==0` and wrong `(base,quote)` revert
   `PriceOracle_NotSupported` (not a 0 or garbage price). Confirm the ctor `NoPlugin`/`ZeroAddress`/
   `ZeroWindow` guards fire. These are tested per the X-Ray — confirm, and probe for a degenerate combo the
   tests miss.
4. **Immutability / re-point.** Params are immutable; re-pointing the vault/window is a redeploy + a router
   `govSetConfig`. Confirm there's no setter/admin/upgrade surface (zero blast radius of its own), and that
   a stale/abandoned clone left wired in the router can't be exploited (the router governance is the gate).
5. **fairTvl() monitoring view.** `fairTvl()` is external. Confirm it's pure-read with no side effect and
   can't be used to grief or to read an inconsistent mid-state.

## What is already proven (don't re-report as new)
`test_getQuote_revert_unsupportedBase`/`_unsupportedQuote`/`_zeroSupply`, `test_ctor_revert_noPlugin`/
`_zeroVault`/`_zeroWindow`. The **lib-side** `NoPlugin`/`BadTimepoints` are NOT directly tested (mock-plugin
gap, per the lib X-Ray §5) — a concrete degenerate-plugin path (length-2-but-meaningless timepoints, or a
stale TWAP that passes the guards) is exactly the gap to target. Frame each finding as fail-open (dangerous,
prices a borrow market) vs fail-closed (safe DoS).
